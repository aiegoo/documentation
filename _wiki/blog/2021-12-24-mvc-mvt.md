## difference

 우선 아키텍처 패턴(Architecture Pattern)은 소프트웨어 내부에 존재하는 구조적인 패턴을 말합니다. 대표적으로 10개 정도가 있습니다.



<p>◆ <span style="background-color: #ffc9af;">아키텍처 패턴</span></p>

-. 계층화 패턴(Layered pattern)

-. 클라이언트-서버 패턴(Client-server pattern) : 우리 주변의 웹 사이트, 온라인 게임 등

-. 마스터 슬레이브 패턴(Master-slave pattern)

-. 파이프-필터 패턴(Pipe-filter pattern)

-. 브로커 패턴(Broker patttern)

-. 피어 투 피어 패턴(Peer-to-peer pattern)

-. 이벤트-버스 패턴(Event-bus pattern)

-. MVC 패턴(Model-View-Controller pattern)

-. 블랙보드 패턴(Blackboard-pattern)

-. 인터프리터 패턴(Interpreter pattern)

Note that MVC is often used by web-frameworks like Laravel, where the received URL requests (maybe made by users) are handled by the Controller and the HTML generated by the View is sent to the client -- So, the View is a part of the backend and the user can never access it directly, and if you experience anywhere the opposite then consider that as an MVC-extension (or even violation). @Panzercrisis, This differs from MVP (like that used in Android OS) where actions route through the View to the Presenter and user have direct access to the View

<p>&nbsp;위의 패턴들 중에서 Django MVT 패턴가 비슷한 <a href="https://hooni40.tistory.com/29" target="_blank" rel="noopener">MVC 패턴</a>에 대해 같단히 정리하고 가겠습니다.(이전에 정리해보았던 MVC 패턴도 확인해주세요!) <b>MVC 패턴</b>은 Model, View, Controller 세 가지로 나눠서 개발하는 패턴입니다. <span style="background-color: #ffc9af;">Model</span>은 <span style="color: #f89009;">데이터를 저장, 보관</span>하고 <span style="background-color: #9feec3;">View</span>는 <span style="color: #009a87;">사용자에게 보여지는 부분</span>을 담당하고 <span style="background-color: #99cefa;">Controller</span>는 <span style="color: #006dd7;">사용자의 입력을 받아 로직을 처리</span>합니다. 이렇게 역할을 분리하여 개발하면 분업이 가능해지고 프로그램 완성도 더 빨리 할 수 있습니다!</p>

<p><b>&nbsp;MVT 패턴</b>은 Model, View, Template를 합친 것인데요, 하나씩 살펴보겠습니다. <span style="background-color: #ffc9af;">Model</span>은 <span style="color: #f89009;">데이터 구조를 생성</span>하고 <span style="color: #f89009;">데이터베이스와 소통</span>하는 역할을 합니다. Model로 데이터 구조를 정리하면 Django는 CRUD를 통해 데이터 베이스와 소통합니다. <span style="background-color: #99cefa;">Template</span>는 이전 시간에 다뤘었는데요, <span style="color: #006dd7;">웹 사이트의 화면 구성을 담당</span>하며 <span style="color: #006dd7;">Template Language를 통해 매번 바뀌는 동적인 화면을 구성</span>합니다. 만약 유튜브를 생각하면 영상, 제목, 댓글은 변경되더라도 <span style="color: #006dd7;">기본적인 틀은 동일</span>합니다. 여기에 Template Language를 이용하면 변하는 세부 내용들도 채워넣을 수 있습니다! 마지막으로 <span style="background-color: #9feec3;">View</span>는 <span style="color: #009a87;">웹 사이트의 로직</span>을 담당하고<span style="color: #009a87;">&nbsp;Model과 Template 사이를 연결</span>해 줍니다. 즉 <span style="color: #009a87;">요청(Request)이 들어오면 요청을 처리하여 응답(Response)</span>해주는 역할을 합니다!</p>

<p><span style="color: #ee2323;">&nbsp;MVC 패턴과 차이점</span>은 <span style="background-color: #ffc9af;">MVC 패턴의 View가 하는 역할</span>을 <span style="background-color: #ffc9af;">MVT 패턴의 Template</span>이 하고 <span style="background-color: #99cefa;">MVC 패턴의 Controller</span>가 하는 역할을 <span style="background-color: #99cefa;">MVT 패턴의 View</span>가 합니다! Django에서 MVT 구조를 하는 이유는 <span style="color: #ee2323;">MVC의 Controller가 했던 역할의 일부를 Django 프레임워크가 직접 처리</span>합니다. 따라서 개발자는 Model, View, Template에 집중하고 나머지는 Django프레임워크가 직접 처리해줍니다.</p>
